# ClipSave — закрытый веб-сервис для скачивания медиа

ClipSave — это приватный веб-сервис, через который можно скачивать видео, аудио и обложки по ссылке.

Сервис подойдёт тем, кому нужен личный удобный инструмент с очередью загрузок, историей скачиваний и поддержкой `cookies.txt` для тех сайтов, где без авторизации видео не скачивается.

## Возможности

- проверка ссылки перед скачиванием;
- скачивание в лучшем качестве;
- выбор формата вручную;
- скачивание в MP3;
- скачивание обложки;
- очередь загрузок с ограничением числа одновременных задач;
- личная история скачиваний для каждого пользователя;
- поддержка личного `cookies.txt` пользователя;
- поддержка общего `cookies.txt` администратора;
- вход по постоянной ссылке;
- одноразовые ссылки-приглашения;
- метки для выданных ссылок, чтобы администратору было удобнее различать доступы;
- возможность в любой момент отозвать доступ у пользователя;
- контроль активности пользователей за последние 5 и 10 минут;
- хранение пользователей, ссылок доступа, сессий и истории в SQLite;
- раздача готовых файлов через Angie из каталога `/download`;
- автоматическая очистка старых файлов из `/download`;
- автоматическое ночное обновление `yt-dlp` и `yt-dlp-ejs`;
- автоустановщик;
- отдельный bash-скрипт для резервного копирования, восстановления и проверки базы SQLite.

## Что есть в репозитории

- `web_ytd.py` — основной backend;
- `templates/` — HTML-шаблоны;
- `static/` — JS и CSS;
- `.env.example` — пример конфигурации;
- `requirements.txt` — зависимости Python;
- `install-versions.env` — переменные для автоустановки;
- `scripts/install.sh` — bash-скрипт установки;
- `scripts/ytd_db.sh` — bash-скрипт для backup / restore / проверки SQLite;
- `deploy/systemd/ytd_web.service` — пример systemd-службы;
- `deploy/angie/site.conf.example` — пример конфигурации сайта для Angie;
- `deploy/angie/download_filename_map.conf` — `map` для корректного имени скачиваемого файла;
- `deploy/angie/00-acme.conf.example` — пример глобальной ACME-настройки для Angie;
- `deploy/cron/crontab.example` — пример системного cron-файла.

## Быстрый старт

Запустите на чистом VPS команду и сервис установится в один клик, от вас потребуется лишь предварительно настроить домен/поддомен на IP адрес сервера:

> Важно: сервис ставится из-под `root` пользователя (создавая нового пользователя для своей работы), поэтому либо зайдите под root, либо временно получите root права через `sudo -i`

```bash
sudo apt update
sudo apt install -y git
git clone https://github.com/OMchik33/Web_YTD.git /root/web-ytd-src
cd /root/web-ytd-src
sudo bash scripts/install.sh
```

## Что делает `install.sh`

Скрипт установки:

- проверяет, что используется Ubuntu 24.04;
- ставит системные пакеты;
- подключает официальный репозиторий Angie и устанавливает Angie;
- создаёт пользователя `botrunner`, папку проекта и каталог `/download`;
- разворачивает проект из локальной папки или из GitHub/Git;
- создаёт виртуальное окружение Python и устанавливает зависимости;
- создаёт `.env`, если его ещё нет;
- подключает systemd-службу;
- создаёт нужные файлы для Angie;
- включает UFW и открывает порты `22`, `80`, `443`;
- создаёт системный cron-файл `/etc/cron.d/ytd_web`;
- настраивает очистку старых файлов из `/download` каждые 5 минут (удаляются загруженные файлы старше 30 минут);
- настраивает обновление `yt-dlp` и `yt-dlp-ejs` в `04:00` по времени сервера.

В конце установки скрипт показывает:

- путь к проекту;
- путь к `.env`;
- обычную ссылку входа;
- админскую ссылку входа;
- команды для проверки службы.

После установки можно проверить состояние сервиса:

```bash
sudo systemctl status ytd_web --no-pager
sudo systemctl status angie --no-pager
sudo ufw status verbose
```

## Если видео не скачивается: как добавить `cookies.txt`

Иногда некоторые сайты не дают скачать видео без авторизации в браузере. В таком случае ClipSave может показать ошибку при попытке загрузки.

Чтобы это исправить, можно загрузить в сервис ваш файл `cookies.txt` от нужного сайта.

### Как получить `cookies.txt`

Установите расширение для браузера:

https://github.com/kairi003/Get-cookies.txt-LOCALLY/

Дальше сделайте так:

- откройте в браузере сайт, с которого хотите скачать видео;
- при необходимости войдите в свой аккаунт на этом сайте;
- нажмите на установленное расширение;
- сохраните cookies в формате **Netscape**.

### Как загрузить файл в ClipSave

В ClipSave в разделе **«Мой cookies.txt»** загрузите сохранённый файл.

После этого попробуйте скачать видео снова.

### Важно

- `cookies.txt` нужен не всегда, а только если сайт без него не отдаёт видео;
- файл должен быть получен именно с того сайта, откуда вы хотите скачать видео;
- если через время скачивание снова перестанет работать, возможно, нужно заново сохранить и загрузить свежий `cookies.txt`.

## Работа с SQLite: backup / restore / проверка

Скрипт `scripts/ytd_db.sh` умеет:

- показывать состояние базы;
- делать резервную копию;
- восстанавливать базу;
- выполнять быструю проверку;
- выполнять полную проверку целостности.

Запуск:

```bash
bash ytd_db.sh
```

Скрипт сам покажет меню на русском языке.

Что важно:

- перед любым восстановлением скрипт автоматически создаёт аварийную резервную копию текущей базы.

## Примечания

- Сервис работает не в корне домена, а по пути `WEB_BASE_PATH`.
- `WEB_PORT` открывать наружу не нужно, потому что FastAPI слушает только `127.0.0.1`.
- SQLite хранит только метаданные. Видео, логи и `cookies.txt` остаются обычными файлами на диске.
- Для YouTube и `yt-dlp-ejs` нужен установленный `node`. Он ставится автоустановщиком.
